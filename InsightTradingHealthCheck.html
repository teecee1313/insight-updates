🩺 Insight Trading — Health Check v2

One tap checks both app channels AND the server. Run it any time — especially after an update goes out.

What this checks

1. The customer app and the 🧪 beta app both load, report their versions, and their code parses with no JavaScript errors.
2. Each live version matches its update.json (catches the "still says the old version" cache problem).
3. The deep-audit integrity markers (🌏×7 · 🎲×5 · 🧊×3) are intact in both builds — a fingerprint that the full, untampered file deployed.
4. The server (data + grading Worker) is up, and reports its features: market-data grades, 🔒 password lock, 📚 history store, and the 🤖 auto-trade bridge queue.

Notes

• This checks that the published files and the server are healthy. It can't see one phone's data-loading (that can be a network moment, not a fault) — so a "bad data day" on one device won't show here.
• If a live version lags after an upload, it's usually GitHub's cache — wait 1–2 minutes and re-run, or hard-refresh.
• The server row needs this page to be on your site (teecee1313.github.io) to work — opened straight from your computer, browsers block that one call; everything else still runs.
• Read-only. No passwords or keys live in this page, and nothing is sent anywhere.
