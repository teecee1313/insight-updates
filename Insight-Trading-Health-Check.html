🩺 Insight Trading — Health Check

Checks your live site and, if you open this next to your files, the local builds too. Run it any time — especially after uploading an update.

What this checks

1. Your live site loads and reports its version.
2. The live version matches what update.json says (catches the "still says the old version" cache problem).
3. Each page's code parses with no JavaScript errors.
4. The core building blocks (app version, key functions) are present.

Notes

• This checks that the files are healthy. It can't see live data-loading on a phone (that depends on the free data proxies and each person's API key), so a "bad data day" isn't a file fault and won't show here.
• If the live version lags after an upload, it's usually a cache — wait 1–2 minutes and re-run, or hard-refresh.
• Runs entirely in your browser. Nothing is sent anywhere.
